The Submission

Two years back I submitted a project about Neural Networks. It was designed to help teach the user about them through actual CoreML examples. Two years later, I feared that I forgot that basic knowledge of which I learnt about. And so, I did something great men before me have done: tried again. Only this time I decided to do something very different. My parents recently got a new iPad. And so instead of making an Xcode playground, I made a Playground Book. The difference here is that one is made on the iPad while the other is made on the computer. There are four chapters to it.

The first one is a brief overview of what it is and lets the user play around with training one. Watching its intelligence grow within seconds.

The next one is about the concept of XOR and how that is used. XOR (exclusive Or) is a logic gate that outputs 1 when exactly one of the two inputs is 1. Here, the user witnesses what kind of data gets used and how it is quickly comprehended.

The third chapter is about Epochs, Learning Rate and Activation Functions. Each concept gets their own page and builds off of each other. Epochs are essentially the number of iterations that are used for pass the training data through the network. More complex DataSets often and number of hidden layers require more epochs. Meanwhile, the learning rate determines how quickly the network changes the weights based on the loss. And lastly, the activation function defines the output of a node for an input value. It can also be used to set the value to a certain range (e.g. between 0 and 1).

The final page is just a conclusion and congratulates the user for their completion.

Journal

Below is what I submitted for their questionnaires. Each had to be answered in 500 words or less which kind of sucked.

List the open source software you used and explain why you used it.

While I did not use open source "software" specifically, I did happen to use open-source data and images for the challenges within my Book. All of the credits can be found within the "Attributes" section at the end of the Book.

Tell us about the features and technologies you used in your Swift playground.

Since my Playground Book is much more visually based and focused on teaching the mechanics of Neural Networks, I was able to build it without the use of any extensive technologies. Let's begin with my use of the basics: Foundation, UIKit, and CoreGraphics. These libraries are seen the most within my Source Files. This is because, as I said before, the program is mostly visually based. However, when I talk about the features side of the project, the implementation of these technologies may seem more impressive. For example, my extensive application of the OOP (Object-Oriented Programming) paradigm. At the base, there are helper extensions and organizational classes, structs, etc. Then on top of that layer are some base classes (like the actual Neural Network) that then get inherited by more specialized classes such as the different ones for the UI and its various needs. That feature was definitely more on the programming side, there are a bunch of features that are also product facing. As for one that stands out on the product side I would say that the Book's use of real data (CSV values of actual Iris flowers and such) to help the user truly grasp what it means to work on Neural Networks was very helpful to the experience. I must add that while my implementation and design of the Neural Network system is a bit primitive compared to the professional ones out there, its application fits very well with this context (teaching about its mechanics). I used a lot of more hands-on animations such as UIBezierPath, CoreAnimation, etc. to handle animating the changes within the brain of the Neural Network. This definitely is needed as it helps to illustrate to the user a lot of the processes that get explained within the Book. An advanced framework that I did use is Apple's Accelerate framework for one of my "base" classes that I wrote about earlier. According to the Apple documentation, "Accelerate provides high-performance, energy-efficient computation on the CPU by leveraging its vector-processing capability." What this means is that it is the framework I need to conduct heavy math. Which heavy math? Interpolation! Within my InterpolationHelper class, all of my methods (that conduct the actual interpolation between Arrays) use this framework. The purpose of this playground was to teach the user the basics of Neural Networks. Basically as an introduction before all of the heavy-hitting Calculus, Linear-Algebra, Penta-byte level computers, and the other hardships that AI developers find themselves navigating through. In a world where we are moving more and more in tune with technology and Artificial Intelligence, we will all find ourselves one day living in a reality where education on this topic can be found and required at schools. Are you ready for that future? In the spirit of community, I made my code open-source and it can be found on GitHub (https://github.com/ozanm/WWDC-2021-Neural-Networks) along with all of my other projects (including my 4 annually previous submissions that were not so lucky with the judging process).
